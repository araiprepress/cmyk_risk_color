# 画像のリスクカラー警告ツール

画像を読み込むと、CMYK変換時に色が大きく変化する可能性があるリスクカラーを検出し、警告表示するWebアプリケーションです。

## 機能

- ドラッグ&ドロップで画像を読み込み
- リスクカラーの自動検出
- リスクカラーを赤で表示した警告画像の生成
- リスクカラーを安全な色に変換した画像の生成
- 画像のダウンロード機能

## 使い方

1. `index.html`をブラウザで開く
2. 画像をドラッグ&ドロップエリアにドロップ
3. リスクカラーの検出結果が表示されます
4. 必要に応じて警告画像や変換後の画像をダウンロード

## ファイル構成

- `index.html` - メインのHTMLファイル
- `style.css` - スタイルシート
- `dd_img.js` - ドラッグ&ドロップ機能
- `risk_color_detector.js` - リスクカラー検出ロジック
- `risky_colors.csv` - リスクカラーのデータベース

## 技術スタック

- HTML5
- CSS3
- JavaScript (Vanilla)
- Canvas API

## Git/GitHub の使い方（学習メモ）

### 今回行った作業の全体像

1. **Gitリポジトリの初期化**
   - `git init` - プロジェクトフォルダをGitで管理できるようにする
   - これにより、変更履歴を追跡できるようになる

2. **SSHキーの設定**
   - `ssh-keygen` - GitHubに安全に接続するための鍵を生成
   - 公開鍵をGitHubに登録することで、パスワードなしでpushできる

3. **ファイルのコミット**
   - `git add .` - 変更したファイルを「ステージング」（コミット準備）
   - `git commit -m "メッセージ"` - 変更を記録（スナップショットを撮る）

4. **GitHubへのアップロード**
   - `git remote add origin URL` - GitHubのリポジトリを「origin」という名前で登録
   - `git push -u origin main` - ローカルの変更をGitHubにアップロード

### よく使うGitコマンド

```bash
# 現在の状態を確認
git status

# 変更を確認（何が変わったか見る）
git diff

# 変更を記録
git add .                    # すべての変更を追加
git add ファイル名            # 特定のファイルだけ追加
git commit -m "変更内容の説明" # 変更を記録

# GitHubにアップロード
git push

# GitHubから最新版を取得
git pull

# 変更履歴を見る
git log
```

### なぜGitを使うのか？

- **バックアップ**: コードをGitHubに保存しておけば、PCが壊れても大丈夫
- **履歴管理**: 過去のバージョンに戻れる
- **共同作業**: 複数人で同じプロジェクトを編集できる
- **実験**: 新しい機能を試して、うまくいかなければ元に戻せる

### 今後の学習ポイント

1. **ブランチ（branch）**: メインのコードを壊さずに実験できる機能
2. **マージ（merge）**: 別々に作った変更を統合する
3. **プルリクエスト（Pull Request）**: 変更をレビューしてから取り込む仕組み
4. **`.gitignore`**: アップロードしたくないファイルを指定する設定ファイル
